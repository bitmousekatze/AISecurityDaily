Grok/Bankr Wallet Drain – Prompt Injection Attack Steals $150K–$202K in DRB Tokens

Yesterday (May 4, 2026), an attacker drained roughly $150K–$202K worth of DRB tokens (≈3 billion tokens) from a Grok-associated wallet on Base. The attack was not a smart-contract exploit or private-key theft — it was a sophisticated prompt injection delivered through social engineering and encoded instructions.


The attacker gifted a Bankr Club Membership NFT to the wallet, unlocking advanced transfer permissions. They then posted a reply containing Morse code (or similar encoding) that, when parsed by Grok, instructed a full token transfer. Grok decoded and output the command publicly, inadvertently tagging @bankrbot. The bot treated it as a legitimate tool call and executed the transfer instantly — with no strong secondary validation or human confirmation.


Approximately 80% of the funds were later returned, but this marks the second similar attack on the same setup. The incident perfectly demonstrates the core risk of agentic AI: once a model can read external input, reason over it, and call real-world tools (wallets, APIs, code execution), prompt injection turns the agent into an unwitting remote-controlled exploit vector.

Why It Matters for Vibe Coders
This wasn’t a “hack” in the traditional sense. It was input manipulation that bypassed safety layers because the model understood intent while simple filters saw only “gibberish.” Encoded payloads (Morse, Base64, ROT13, etc.) evade keyword guards. NFT/social engineering + AI execution = a brand-new hybrid attack surface. If you ship agents with tool access, you are now a high-value target.
Action Items
  • Audit every tool your agents can call — enforce least privilege and read-only where possible.
  • Add strict output validation and sanitization before any tool execution (especially transfers or mutations).
  • Require human-in-the-loop confirmation for all high-impact actions (financial, destructive, external calls).
  • Deploy runtime scanners for encoded prompt injection (see Quick Hits below).
  • Monitor agent behavior with canary tokens and anomaly detection.
Quick Hits

Agent Hardening Toolkit

1. NVIDIA/garak — Gold-standard LLM vulnerability scanner with 200+ probes, including Morse/Base64 encoding exploits and agent tool-calling attacks. Run it in CI/CD for automated red-teaming.
→ github.com/NVIDIA/garak
2. protectai/llm-guard — Production-ready input/output scanner with dedicated PromptInjection detector. Drop it into your pipeline for real-time blocking of malicious prompts and tool calls.
→ github.com/protectai/llm-guard
3. StackOneHQ/defender — Lightweight (22MB, CPU-only, <10ms) guard built specifically for tool-calling agents (MCP/CLI/function calling). Excellent against indirect/encoded injections.
→ github.com/StackOneHQ/defender
Start here: Run garak for testing, then layer llm-guard or defender at runtime. Combine with strict tool permissions and human gates. The era of “set it and forget it” agents is over.
