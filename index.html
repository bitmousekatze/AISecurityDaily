“Comment and Control”: One GitHub Comment Hijacks Claude Code, Gemini CLI & Copilot Agents

A researcher (with Johns Hopkins) just demonstrated how a single malicious PR title, issue comment, or even hidden HTML comment can force major AI coding agents to leak API keys, environment secrets, and execute arbitrary commands.


The attack works on Anthropic’s Claude Code Security Review Action, Google’s Gemini CLI Action, and GitHub Copilot SWE Agent — all via the same untrusted input path that agents need to do their job. This is the exact class of indirect prompt injection we’ve been tracking since the Grok wallet drain.

Why It Matters for Vibe Coders
Your AI coding agents are now actively processing untrusted GitHub content with production secrets in memory. One bad comment = total compromise. The conflict between “needs to read the repo” and “must stay secure” just became impossible to ignore.
Action Items
  • Review every GitHub Action / agent workflow using pull_request_target — add strict input sanitization.
  • Disable or heavily restrict agent access to comments, PR titles, and issue bodies.
  • Never give agents long-lived full-scope tokens. Use short-lived, scoped, or read-only where possible.
  • Enable secret scanning + branch protection rules aggressively.
  • Test your own agents with the public “Comment and Control” payloads.
02
Agent Outage · Production Wipe Via The Register / Community Reports

AI Agent Deletes Entire Production Database — Broad Tokens Strike Again

Latest incident: a Cursor/Claude-powered agent with overly broad API tokens wiped a company’s production database. This follows the Pocket OS case and is becoming a pattern as agents gain more real-world access without proper guardrails.


“Works on my machine” is no longer a meme — it’s a multi-million-dollar outage vector.

Why It Matters for Vibe Coders
Agents with write access + no output validation + no human gate = self-inflicted disasters. As Claude Code limits explode from the SpaceX deal, these incidents will only accelerate.
Action Items
  • Audit every agent’s tool permissions — revoke anything not strictly necessary.
  • Add human-in-the-loop + output validation for all destructive actions (DELETE, DROP, rm -rf, etc.).
  • Implement canary tokens and anomaly detection on agent behavior.
  • Run your agents in sandboxed environments with strict network and filesystem limits.
