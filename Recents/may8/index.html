ClaudeBleed: Claude Chrome Extension Allows Remote AI Takeover via Prompt Injection

A high-severity vulnerability in the official Claude Chrome extension (ClaudeBleed) lets attackers inject malicious instructions remotely. Once triggered, the compromised agent can exfiltrate data from connected services including Gmail, GitHub, and Google Drive.


The attack requires no user interaction beyond visiting a page that triggers the extension — classic indirect prompt injection against browser-based agents.

Why It Matters for Vibe Coders
Browser extensions are the new always-on agents. If you use Claude (or any AI coding tool) in Chrome, your entire workspace is one malicious site away from compromise. The extension has full access to every tab and connected account.
Action Items
  • Disable the Claude Chrome extension until patched (or use in isolated profile).
  • Audit all AI browser extensions (Claude, Cursor, Gemini, etc.) for prompt injection surfaces.
  • Never let browser-based agents have persistent credentials in memory.
  • Use containerized or sandboxed browser profiles for AI tooling.
  • Monitor extension permissions — revoke anything accessing sensitive sites.

Comment & Control Attacks Now in the Wild — Multiple Orgs Hit

The GitHub comment/PR-based prompt injection technique that simultaneously owns Claude Code, Gemini CLI, and GitHub Copilot is seeing active exploitation. Researchers and defenders are reporting real secret leaks and unauthorized actions in the last 24 hours.


Same vector we covered yesterday — now production reality.

Why It Matters for Vibe Coders
Every repo using AI agents that read comments or PRs is now live fire. The “just let the agent see the context” assumption is broken.
Action Items
  • Immediately review and sanitize all GitHub Actions using pull_request_target.
  • Block or heavily filter PR titles, comments, and issue bodies before agent ingestion.
  • Rotate any leaked tokens/secrets from affected workflows.
  • Test your own agents against the public Comment & Control payloads.
