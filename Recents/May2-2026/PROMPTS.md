# AI Vibe Security Scanner — Jump-Start Prompts

Six ready-to-use prompts that activate the security scanner for the most common
use cases. Copy, paste, and swap the placeholders for your actual content.

---

## Prompt 1 — Full code scan (paste and check)

Use when: you've just generated code with an AI assistant and want a complete
security audit before committing or shipping.

```
I generated the following code using [Claude / Cursor / Copilot / ChatGPT].
Please run a full AI vibe security scan on it and give me a structured report.
Check for prompt injection patterns, hallucinated packages, hardcoded secrets,
OWASP Top 10 vulnerabilities, and anything else that looks risky for
AI-generated code specifically.

[PASTE CODE HERE]
```

## Prompt 2 — MCP server and agent config audit

Use when: you're setting up or reviewing an MCP server configuration, a
`.cursorrules` file, a `CLAUDE.md`, or any other agent instruction file.

```
Please audit this agent configuration file for security issues. I'm specifically
worried about prompt injection in tool descriptions, over-permissioned tool
access, MCP servers loading from untrusted sources, and any instructions that
could be used to manipulate an AI agent reading this file.

[PASTE CONFIG FILE CONTENTS HERE]

Also tell me: are there any tool descriptions that could be misread as
instructions to bypass safety checks or override system behavior?
```

---

## Prompt 3 — Dependency and package hallucination check

Use when: you have a `package.json`, `requirements.txt`, `Cargo.toml`,
`go.mod`, or any dependency file generated by an AI tool and want to catch
slopsquatted or hallucinated package names before installing.

```
Here is a dependency file generated by an AI coding assistant. Please check
every package name for signs of hallucination or slopsquatting — packages
that look real but don't exist, or names that are slightly wrong versions of
legitimate packages that attackers could register as malware.

Flag anything you can't confidently verify and tell me which registry to
check for each flagged package.

[PASTE DEPENDENCY FILE HERE]
```

## Prompt 4 — CI/CD and GitHub Actions agent permission audit

Use when: an AI agent (Claude Code, Copilot, Cursor, etc.) has been added to
your CI/CD pipeline and you want to check that it doesn't have excessive
permissions or expose secrets to the agent runtime.

```
Please audit this CI/CD configuration for AI agent security issues. Focus on:

1. Whether GITHUB_TOKEN or other secrets have overly broad scopes
2. Whether the AI agent step has access to production credentials it shouldn't
3. Whether there's a human approval gate before agent-triggered deployments
4. Whether auto-merge or auto-deploy can be triggered by agent output alone
5. Any prompt injection risk in workflow names, PR titles, or commit messages
   that get passed to the agent as context

[PASTE GITHUB ACTIONS YAML OR CI CONFIG HERE]
```

## Prompt 5 — Targeted OWASP scan for a specific vulnerability class

Use when: you have a hunch about a specific vulnerability type, or a previous
scan flagged something and you want to go deeper on that one category.

```
I need a focused security review of this code specifically for [CHOOSE ONE:
SQL injection / XSS / command injection / path traversal / SSRF / 
authentication bypass / IDOR / insecure deserialization].

This code was AI-generated so pay extra attention to input sanitisation —
LLMs tend to skip it when optimising for working examples.

For every instance you find, show me:
- The exact vulnerable line(s)
- A concrete attack example showing how it could be exploited
- The fixed version of the code

[PASTE CODE HERE]
```

## Prompt 6 — Repo-level threat model for a vibe-coded project

Use when: you want a high-level threat model for an entire AI-generated
project, not just a single file — useful before a security review, before
going to production, or before open-sourcing.

```
I built this project primarily using AI coding assistants. Please create a
threat model for it based on the structure and code I'll share below.

Specifically I want to understand:
1. What are the highest-risk attack surfaces given this is AI-generated code?
2. Which of the six AI vibe security scan modules apply most to this project?
3. What security controls are visibly absent that I should add before launch?
4. Are there any patterns in the code that suggest the AI assistant introduced
   systematic vulnerabilities (e.g. consistently missing input sanitisation,
   always using string concatenation for queries)?

Project description: [BRIEF DESCRIPTION OF WHAT IT DOES]
Tech stack: [LANGUAGES, FRAMEWORKS, DATABASES]

[PASTE KEY FILES, DIRECTORY STRUCTURE, OR REPRESENTATIVE CODE SAMPLES]
```

## Quick reference — which prompt for which situation

| Situation | Prompt to use |
|-----------|--------------|
| Just generated some code, quick check | Prompt 1 |
| Setting up Claude Code / Cursor / MCP | Prompt 2 |
| `npm install` / `pip install` from AI output | Prompt 3 |
| AI agent in GitHub Actions or CI | Prompt 4 |
| Specific vuln class investigation | Prompt 5 |
| Full project before launch / open-source | Prompt 6 |

---

## Tips for getting the best results

**Include context about the AI tool used.** Different tools have different
failure patterns. Cursor tends to miss auth checks; Copilot tends to hallucinate
package names; Claude Code tends to be more verbose about what it's doing.

**Run Prompt 1 on every file before committing.** The earlier you catch it,
the cheaper it is to fix.

**Use Prompt 3 before every `npm install` or `pip install`.** Slopsquatting
attacks are silent — the malicious package installs without error.

**Don't skip the MCP audit (Prompt 2).** MCP tool descriptions are a new
injection surface almost no one is checking yet.

**For Prompt 6, paste your actual code.** A threat model based on a description
alone will miss the specific patterns your AI assistant introduced.
