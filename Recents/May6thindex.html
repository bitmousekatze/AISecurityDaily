Critical Exposure Via The Hacker News / Intruder Research

We Scanned 1 Million Exposed AI Services — The Security Nightmare Is Worse Than Anyone Expected

In the wake of the ClawdBot fiasco (averaging 2.6 CVEs per day), Intruder used certificate transparency logs to scan over 1 million AI-related services. The findings are brutal: AI infrastructure is more vulnerable, exposed, and misconfigured than any category they've ever tested.


Key red flags: 31% of exposed Ollama APIs responded to unauthenticated prompts. No default authentication on LLM gateways, agent platforms leaking credentials, chatbots exposing conversation history, and self-hosted setups with wide-open APIs. The rush to deploy AI is completely outpacing basic security hygiene.


This isn't theoretical — these are live, internet-facing services powering real agentic workflows. Attackers don't need zero-days when the front door is unlocked.

Why It Matters for Vibe Coders
If you're self-hosting Ollama, LangChain, local agents, MCP servers, or any AI tooling in production (or even dev), you are now part of the visible attack surface. "It works on my machine" no longer cuts it when your AI infra is public. The window between deployment and exploitation has collapsed.
Action Items
  • Inventory every exposed AI endpoint (Ollama, vLLM, LM Studio, custom agents) with curl or Intruder-style scanning.
  • Enforce API keys + mTLS / network-level isolation immediately — no unauthenticated LLM calls.
  • Run automated external scans on your own infra (garak + custom probes).
  • Implement strict auth + rate limiting on all MCP / tool-calling endpoints.
  • Assume every self-hosted AI service is public until proven otherwise.
02
MCP · High Via X / GitHub Advisories

eBay API MCP Server CVE-2026-27203: Environment Variable Injection = Instant RCE

CVE-2026-27203CVSS ~8.3


The open-source eBay API MCP Server (used by AI agents for real-time marketplace actions) is vulnerable to environment variable injection via the ebay_set_user_tokens tool. The updateEnvFile function blindly writes attacker-controlled values into .env — enabling full RCE and credential theft.


This is the latest in a growing wave of MCP/protocol-level flaws that turn agent tool-calling into remote code execution. If your agents touch external APIs via MCP, this class of bug is now table stakes.

Why It Matters for Vibe Coders
Agentic AI doesn't just read data — it mutates real systems. Any tool that writes config/files without strict validation becomes a backdoor. MCP servers are the new "cloud metadata service" for attackers.
Action Items
  • Patch or fork any MCP servers you run (especially eBay, Shopify, etc.).
  • Sandbox tool execution — never let agents write to .env, config files, or host filesystem directly.
  • Validate + sanitize every parameter before it touches the shell or filesystem.
  • Use read-only tool scopes wherever possible.
